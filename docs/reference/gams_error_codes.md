# GAMS error codes

:::{note}
**In brief** — A quick-reference guide to the GAMS errors most commonly seen when running OPEN-PROM, grouped by phase (compilation, pre-triangular, infeasibility, licensing), with the likely cause and fix for each.
:::

Encountering errors while working with GAMS is common, but knowing how to troubleshoot them saves time. This page summarises the errors most frequently met when running OPEN-PROM, organised by the phase in which they arise: compilation, pre-triangular infeasibilities, infeasibilities, and licensing.

## Compilation errors

Compilation errors occur during the model's compilation phase. To locate them, search for four asterisks (`****`) in the `.lst` file generated by GAMS.

:::{tip}
The `****` marker always sits at the start of the line where GAMS flags a problem, so a plain text search through the `.lst` file is the fastest way to jump to the offending statement.
:::

| Code | Message | Cause | Fix |
|---|---|---|---|
| **101** | Syntax error, unexpected keyword in the declaration of a component | A syntax error in the GAMS code. | Double-check the syntax and make sure all keywords are spelled correctly. |
| **102** | Symbol `xxx` already defined | A symbol is defined more than once. | Check for duplicate definitions of symbols (variables, equations, parameters) and ensure each is defined only once. |
| **103** | Inconsistent equation definition | Inconsistencies in the definition of equations. | Review the equations and ensure they are correctly defined for the problem at hand. |

## Pre-triangular infeasibilities

| Code | Message | Cause | Fix |
|---|---|---|---|
| **170** | Pre-triangular infeasible or unbounded | The initial values supplied for variables or parameters lead to infeasibility or unboundedness. | Adjust the initial values or the constraints to ensure feasibility. |
| **171** | GAMS: internal solver error | An internal solver error. | Increase the solver tolerance or switch to a different solver; if the problem persists, contact GAMS support. |

## Infeasibilities

| Code | Message | Cause | Fix |
|---|---|---|---|
| **201** | Model infeasible | No feasible solution satisfies all constraints. | Review the model constraints and objective function to find conflicting or overly restrictive conditions. |
| **202** | Constraint `xxx` is infeasible | A specific constraint cannot be satisfied. | Inspect the constraints linked to the given identifier (`xxx`) and revise them as needed. |

## License errors

**"The installed license is invalid" / "License has expired".** Ensure that the `gamslice.txt` file inside the installation `bin` folder is correct and holds valid license information.

:::{warning}
GAMS does not always read the license file from the `bin` folder. Check the file path quoted in the error message to confirm which license file GAMS is actually referencing.
:::

## General troubleshooting tips

- **Check log files.** Review the GAMS log files for detailed error messages and diagnostic information.
- **Consult the documentation.** The GAMS documentation and user guides explain error messages and offer troubleshooting advice.
- **Update the GAMS version.** Newer versions may include bug fixes and improvements relevant to the error.
- **Search forums and communities.** Solutions to common GAMS errors are often found in online forums; otherwise, post your question to seek help from experienced users.

:::
